Einzelplatz / Client

Bildschirm

Es wird ein Mehr-Monitor-System unterstützt.
Monitor mit mindestens 1280 x 1024 Bildpunkten
(für übersichtlichen Mehr-Fensterbetrieb empfehlen wir 1920 x 1080 und mehr).
Windows: Für den Mehr-Fensterbetrieb muss das MDI-Fenster (epirent- Hauptfenster) über mehrere Monitore beim Start manuell erweitert werden.

Schnittstellen

Einige Module benötigen für den Einsatz weiterer Zusatzhardware USB-Anschlüsse
am Einzelplatz / Clientrechner (Barcodescanner, ggf. Scanner und Drucker).

Speicherplatz

Die Anwendung inkl. dem zwischengespeicherten Cache nehmen ca. 1 GB ein. Für einen Einzelplatz (ohne Server) sollten für die Datenbank und die in epirent gespeicherten Fremddokumente (Datei-Ablage) zusätzlich 10 GB eingerechnet werden.


Server

Mehrplatzbetrieb

Ein eigenständiger Serverrechner wird empfohlen.
Eine virtuelle Maschine ist nur mit eigener SSD für die VM empfehlenswert.

Heterogener Netzwerkbetrieb ist möglich (Arbeitsplätze Windows und macOS gemischt, gleichgültig ob der Server mit Windows oder Mac OS X betrieben wird).

Mindestens 100MBit-Netzwerk, besser GBit-Netzwerk.

WLAN ist nur bei garantiert dauerhaft stabiler und guter Signalverbindung ohne Unterbrechungen (auch nicht kurzzeitig) akzeptabel.

epirent Arbeitsplätze sind Terminalserver kompatibel (auch Citrix). Pro gleichzeitig arbeitendem Anwender sind ca. 0,75 GB RAM empfohlen. Die Server Anwendung kann nicht auf dem selben Terminalserver betrieben werden. Es werden keine Microsoft Lizenzen für den Betrieb von epirent vorausgesetzt.

Anzahl der Prozessoren

epirent kann mehrere Prozessoren / Prozessorkerne gleichzeitig nutzen.

Je nach Anforderung können also mehrerer Prozessoren / Prozessorkerne im Serverrechner sinnvoll sein, um gleichzeitige Rechen-, Such- oder Auswertungsprozesse schneller verarbeiten zu können.

Hier zählen nur echte Kerne, keine virtualisierten Kerne (wie Hyper-Threading-Kerne oder virtualisierte Kerne in einer VM). Eine Aufteilung auf virtualisierte Kerne bringt keine Verbesserung der Performance, sondern verursacht einen höheren Verwaltungsaufwand und kann somit die Performance nachteilig beeinträchtigen.

Speicherplatz

Für das Windows oder Mac Betriebssystem sind 10 GB freizuhalten.
Für höchste Datensicherheit und schnelle Zugriffszeiten empfehlen wir epirent und die Datenbankdatei auf einer eigenen Festplatte (SSD) zu betreiben.
Bei großen Unternehmen (15 Clients und mehr) können in z.B. 5 Jahren mehrere 100 GB Speicherplatz benötigt werden. Dazu kommen zusätzlich Fremddokumente (PDF's, Bilder, etc.).
Für einen funktionsfähigen Reparatur-/Wiederherstellungs-/Rollback-Mechanismus sollte stets mindestens die doppelte Grösse der Datenbankdatei als freier Speicherplatz zur Verfügung stehen.

Festplatten

System mit Serverfestplatte („24/7“) bzw. SSD empfohlen. Eine zusätzliche SSD zur Ablage der Fremddokumente ist erwägenswert.

Zur erststufigen Backup-Erstellung kann eine zusätzliche (ggf. externe) HDD empfehlenswert sein.

Eine anschließende Sicherung z.B. per RAID ist sinnvoll.


Datenbank

Relationale Datenbank 4D mit integriertem Web- und Anwendungsserver (im Lieferumfang enthalten)
